# Human-in-the-Loop 체크포인트

## 개요
초보자가 Senior-level 결과물을 만들기 위해 Human-in-the-Loop (HitL) 체크포인트를 설정합니다. AI가 자율적으로 작업하되, 정해진 단계에서 반드시 인간의 승인을 받습니다.

## 체크포인트 유형

### 1. 시작 체크포인트 (Start Gate)
**조건:** 새로운 연구课题 시작 전
**질문:**
-研究方向가 적절한가?
-예상 결과물이 실용적인가?

### 2. 문헌 조사 체크포인트 (Literature Review Gate)
**조건:** 1차 문헌 조사 완료 후
**질문:**
-발견한文献이 충분한가?
-주요 논문 3개 이상이 포함되었는가?
-보완 조사 필요한 영역이 있는가?

### 3. 설계 체크포인트 (Design Gate)
**조건:** 방법론/아키텍처 설계 완료 후
**질문:**
-설계가 기존 연구를 기반으로 하는가?
-구현 가능한 범위인가?
-잠재적 위험 요소가 있는가?

### 4. 구현 체크포인트 (Implementation Gate)
**조건:** 주요 구현 완료 후
**질문:**
-코드 품질 기준 충족하는가?
-테스트가 충분한가?
-문서화가 완료되었는가?

### 5. 분석 체크포인트 (Analysis Gate)
**조건:** 결과 분석 완료 후
**질문:**
-결론이 데이터에 기반하는가?
-오류 가능성을 검토했는가?
-대안 해석을 고려했는가?

### 6. 완료 체크포인트 (Completion Gate)
**조건:** 최종 보고서 작성 완료 후
**질문:**
-모든 요구사항을 충족했는가?
-추가 연구 필요 영역이 있는가?
-결과를 어떻게 적용할 계획인가?

## 자동화 규칙

### 승인 대기 상태
체크포인트에서 보류 시:
1. AI는 현재 상태를 Wiki에 저장
2. Discord로 알림 전송
3. 사용자 응답 대기

### 승인 시
1. AI는 다음 단계로 진행
2. 진행 상황을 Wiki에 기록
3. 필요시 하위 단계 체크포인트 건너뛰기 가능

### 수정 요청 시
1. AI는 피드백을 분석
2. 수정 계획 제시
3. 수정 후 재승인 요청

## 핵심 원칙

1. **빈도:** 프로젝트 규모에 따라 3-6개 체크포인트 권장
2. **유연성:** 각 체크포인트는 상황에 따라 조정 가능
3. **기록:** 모든 결정과 피드백을 Wiki에 기록
4. **자동화:** 반복적인 확인은 Cron으로 자동화
