# 기획 노트와 세션 로그

## Definition
Obsidian 기반 프로젝트 기억 시스템의 두 가지 핵심 문서. **시작 전**策划 노트, **매 세션 후** 세션 로그.

## 기획 노트 (Project Overview)

**목적: AI와 미래의 나를 위한 설명서**
- "나는 이런 걸 만들고 싶고, 이런 기능이 필요하고, 이런 기술을 쓰고 싶어"
- AI가 정확한 결과물을 내놓도록 하는 것

**핵심 네 가지:**
1. **프로젝트 목적** — 뭘 만들고 싶은지
2. **기술 스택** — 어떤 기술을 쓸지
3. **파일 구조** — 어떤 파일을 만들지
4. **현재 상태** — 어디까지 했는지

**실수 방지:**
- 너무 자세하게 쓰려고 하지 말 것 → "대충 이런 느낌"이면 충분
- 기술 스택 결정이 안 되면 일단 하나 고름 → 나중에 바꿀 수 있음
- 기획 노트 없이 코딩하지 말 것 → 3일이면 다 잊어버림

---

## 세션 로그 (Session Log)

**목적: "오늘 뭘 했는지"를 기록하여 다음 세션의 출발점**

5개 항목, 작성 시간: 3분
1. 시작 전 상태
2. 오늘 완료한 것
3. 다음에 할 일
4. 발생した 에러 (如果有)
5. 알게 된 것 (新技术, 패턴等)

**CC에게 맡기기:**
```
"이 세션 로그를 Obsidian에 저장해줘"
```
CC가 세션 동안 한 작업을 기억하고 있어 꽤 정확한 로그를 생성.

## Obsidian 폴더 구조 예시

```
프로젝트/
└── 할일앱/
    ├── 프로젝트-개요.md      # 기획 노트
    ├── 체크리스트.md         # 현재 상태/진행도
    ├── 설계/
    │   ├── 아키텍처.md
    │   └── Phase3-지시서.md
    ├── 로그/
    │   ├── 세션로그/
    │   │   └── 2026-04-25.md
    │   └── 에러로그.md
    └── 의사결정.md           # 왜 이 기술을 선택했는지
```

**미래의 나를 위한 안내판 역할:**
- 3일 뒤에 이 폴더를 열면 "아, 내가 이걸 만들고 있었지"
- AI에게 "이 폴더 전부 읽어봐" → 5초 만에 맥락 전달
