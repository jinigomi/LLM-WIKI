# 바이브코딩 — 책 개요

**원서**: 박경태, *바이브코딩*, wikidocs

## 핵심 문제

AI 코딩 도구가 처음이라면: "터미널을 열고 '할일 관리 앱 만들어줘'라고 입력한다. Claude Code가 코드를 쏟아낸다. 뭔가 돌아간다. 기분이 좋다. **그리고 3일 뒤에 돌아오면 — '이게 뭐였지?'**"

## 핵심 원칙: 기획 노트 우선

코드를 치기 **전에** 노트를 먼저 쓴다.

**이유**:
1. AI는 내가 뭘 만들고 싶은지 모른다
2. "대충 이런 느낌"이면 충분 — 디테일은 Claude와 대화하면서 채움
3. 미래의 나를 위한 것 — 3일 뒤에도 "아, 내가 이걸 만들고 있었지" 파악 가능

## 도구 연결 (MCP)

MCP 연결 없이 Claude Desktop과 대화하려면 복사-붙여넣기 필요. MCP 연결 시 Claude가 Obsidian 파일을 직접 읽음 → 복사-붙여넣기 사라짐.

→ [[concepts/mcp-프로토콜]]

## AI 협업 역할 분배

| 도구 | 역할 |
|------|------|
| Claude Desktop | 설계 대화, 기획, 설계 확정 |
| Claude Code | 실제 코딩, 구현 |
| Obsidian | 기억 저장소, 프로젝트 맥락 유지 |

→ [[concepts/오케스트레이션]]

## 설계 대화 3단계

1. 기획 노트를 읽게 한다
2. 설계를 요청한다
3. 결과를 Obsidian에 저장한다

## 설계 원칙

- Claude의 제안은 **출발점이지 최종 결정이 아니다**
- 이해하지 못한 설계로 코딩하면 나중에 반드시 문제 발생
- 70%만 확정되면 구현 시작, 나머지 30%는 만들면서 채움

## "기억하는 개발" 핵심

> 대화창은 흘러가는 물이고, Obsidian은 저수지다. **중요한 것은 저수지에 담아둬야 한다.**
