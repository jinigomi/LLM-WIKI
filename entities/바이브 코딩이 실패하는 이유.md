# 챕터별 요약 — 바이브코딩

## Part 1: 도구 연결

### Ch.7: MCP로 세 도구 연결하기
MCP(MCP 프로토콜)로 Claude Desktop ↔ Obsidian ↔ Claude Code를 연결.
복사-붙여넣기 없이 파일을 직접 읽고 쓸 수 있게 됨.

## Part 2: 워크플로우

### Ch.9: 프로젝트를 시작하기 전에 노트부터 쓴다
- 코딩 전에 기획 노트 작성
- 기획 노트 = AI에게 주는 설명서 + 미래의 나를 위한 기억
- 핵심 4가지: 목적, 기능, 기술, 파일 구조

### Ch.10: Claude Desktop과 대화하며 설계를 잡는다
- 설계 대화 3단계: 읽게하기 → 요청 → Obsidian 저장
- Claude의 제안은 출발점, 최종 결정은 나
- 70%만 확정되면 구현 시작

### Ch.11: Claude Code에게 구현을 맡긴다
- 설계를 넘기고 Claude Code가 코딩
- [[오케스트레이션]] 패턴 적용

### Ch.12: 결과를 다시 노트에 기록한다
- 구현 후 Obsidian에 기록하는 습관
- 체크리스트/세션로그 업데이트

### Ch.13: 이 사이클이 왜 마법처럼 작동하는가
- 노트 → 설계 → 구현 → 기록 사이클
- [[기억하는-개발]] 패턴의 핵심

## Part 3: 반복과 회고
