## 5단계 프레임워크

### 1. Framing (문제 정의)
- 무엇을 해결해야 하는가?
- 성공 기준은 어떻게 정의하는가?

### 2. Discovery (정보 수집)
- 어떤 제약조건이 있는가?
- 기존에 비슷한 문제가 있었는가?

### 3. Architecture (구조 설계)
- 큰 그림에서 시작
- 오케스트레이션 패턴 적용

### 4. Roadmap (로드맵)
- 작업 분해
- 마일스톤 설정

### 5. Spec (상세 사양)
- Task Spec + ADR 통합
- 코딩 에이전트가 직접 실행 가능한 형식
