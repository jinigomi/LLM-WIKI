# APP_BLUEPRINT.md 생성 메타프롬프트

> @DeRonin_이 제작하고 @lucas_flatwhite가 공유한, "vibe coding에서 가장 중요한 단 하나의 프롬프트". 코드를 쓰기 전에 제품/시장/아키텍처/출시 전략을 한 번에 구조화하는 AI 협업 블루프린트.

## Context

앱 개발을 시작하기 전에 AI에게 투입하는 **메타프롬프트**다. "너는 시니어 풀스택 엔지니어"라는 페르소나를 부여한 후, 코드 한 줄 쓰기 전에 완전한 `APP_BLUEPRINT.md`를 생성하게 한다. 핵심은 **프로덕션 수준, 지름길 없음**.

## 원문

```
당신은 수익성 있는 SaaS 앱을 10개 이상 구축해본 시니어 풀스택 엔지니어입니다.

현대적인 아키텍처, 클린 코드 패턴, 확장 가능한 인프라를 잘 이해하고 있습니다.

저는 지금 앱을 만들고 있으며, 코드 한 줄을 작성하기 전에 완성도 높은 APP_BLUEPRINT.md를 먼저 만들어야 합니다.

제가 필요한 내용은 다음과 같습니다:

## 시장 검증:
- 정확히 누가 이 제품에 비용을 지불하는가? 고객 세그먼트 3가지를 제시할 것
- 이들이 현재 대체 솔루션에 얼마를 지불하고 있는가? 정확한 금액으로 제시할 것
- 왜 이 제품으로 전환할 것인가? 기존 솔루션의 문제점은 무엇인가?
- 출처를 포함한 TAM/SAM/SOM 분석

## 경쟁 환경:
- 상위 경쟁사 5곳과 정확한 가격 플랜
- 실제 사용자 리뷰 기반의 주요 약점
- 이 제품이 메우는 정확한 시장 공백
- 경쟁사의 랜딩 페이지와 가격 페이지 링크

## 수익화 모델:
- 정확한 금액이 포함된 가격 플랜 설계 (Free / Pro / Enterprise)
- 고객 수 10명, 50명, 200명, 1,000명 기준 매출 예상
- 결제 제공업체 구현 방식 (Stripe / Lemon Squeezy)
- 무료 체험 기간과 전환율 가정

## MVP 기능 범위:
- 핵심 기능은 3~5개만. 그 이상은 제외
- 각 기능별 사용자 스토리, 인수 조건, 구현 복잡도
- V1에서 명시적으로 만들지 않을 것
- 사용자가 무료 플랜에 머무르지 않고 결제하게 만드는 핵심 기능 1가지

## 기술 스택 결정:
- 프론트엔드, 백엔드, 데이터베이스, 호스팅, AI API
- 각 선택지에 대해 다른 대안보다 이것을 선택한 이유
- 사용자 수 0명 / 100명 / 1,000명 / 10,000명 기준 월간 비용 분석
- 인증 방식 선택 (Clerk / Supabase / Firebase)

## 데이터베이스 스키마:
- 전체 스키마: 모든 테이블, 관계, 인덱스
- Row-level security 정책
- 마이그레이션 파일 구조

## API 아키텍처:
- 모든 엔드포인트: 메서드, 경로, 요청/응답 스키마
- 엔드포인트별 인증 요구사항과 rate limit
- 필요한 경우 webhook 엔드포인트

## 사용자 플로우 & 화면:
- 전체 여정: 랜딩 > 회원가입 > 온보딩 > 핵심 기능 > 결제
- 각 화면의 목적과 주요 컴포넌트
- 빈 상태, 로딩 상태, 에러 상태

## 출시 체크리스트:
- 분석 도구 (PostHog / Mixpanel), 에러 모니터링 (Sentry)
- SEO 메타 태그, OG 이미지, 사이트맵
- 트랜잭션 이메일 (환영 이메일, 영수증, 이탈 방지)
- 법적 페이지 (이용약관, 개인정보처리방침), DNS 설정

## 배포 및 성장 전략:
- 첫 100명의 사용자 확보를 위한 구체적인 채널과 전술
- X, Product Hunt, Reddit, Indie Hackers용 콘텐츠 계획
- 제품 내부에 설계된 자연 성장 루프
- 추천 메커니즘과 바이럴 계수 목표

## 구축 순서:
- 1주차: 무엇을 출시하고, 무엇이 배포 가능한 상태가 되는가
- 2주차: 무엇을 출시하고, 무엇이 배포 가능한 상태가 되는가
- 3주차: 무엇을 출시하고, 무엇이 배포 가능한 상태가 되는가
- 4주차: 출시 준비 완료 마일스톤

제가 앱 아이디어를 설명하면, 아래 내용을 포함한 완전한 블루프린트를 생성해주세요:

1. 위 모든 섹션이 포함된 전체 APP_BLUEPRINT.md
2. 모든 디렉터리가 포함된 프로젝트 폴더 구조
3. 데이터베이스 스키마 SQL 마이그레이션 파일
4. API 라우트 정의
5. 환경변수 템플릿 (.env.example)
6. 지속적인 개발 컨텍스트를 위한 CLAUDE.md
7. 배포 가능한 마일스톤이 포함된 주차별 구축 계획

모든 내용은 튜토리얼이 아니라 프로덕션 수준으로 작성해주세요.
실제 에러 핸들링, 적절한 로깅, 이후 확장 가능한 패턴을 포함해주세요.
지름길은 사용하지 마세요.
```

## 분석

11개 섹션이 커버하는 범위:
- **시장** (Market Validation, Competitive Landscape, Monetization)
- **제품** (MVP Feature Scope, User Flows & Screens)
- **기술** (Tech Stack, DB Schema, API Architecture)
- **출시** (Launch Checklist, Distribution & Growth)
- **실행** (Build Sequence — 주차별 마일스톤)

코드를 쓰기 **전에** 모든 걸 정리한다는 점에서 [[plan-before-code]] 패턴의 결정판.
