# vibe-coding-beginner-methodology — 소스 비교 분석

> NotebookLM으로 6개 바이브 코딩 가이드 소스를 분석하여 주요 축별로 정리한 비교.

## 📊 소스 간 특성 비교

| 소스 | 대상 | 강점 | 독특한 관점 |
|------|------|------|-------------|
| codingwithvibe.com | 초보자~중급 | PRD, 와이어프레임, 3계층 프롬프트 | "계획 먼저, 코드는 나중에" |
| softr.io | 비개발자 | 노코드+AI 결합, 시각적 빌더 | 기술 스택 선택 가이드 |
| interviewkickstart.com | 취업준비생 | 보안(62% 취약점), AI Slop 경고 | "무엇이 잘못될 수 있을까?" 질문 |
| aitoolsclub.com | 초보자~프로 | 도구 분류 체계적, 4유형 | 에이전트 vs IDE 통합 구분 |
| GitHub awesome-vibe-coding | 개발자 | 큐레이션, 리소스 링크 | 커뮤니티 지식 집합 |
| datacamp.com | 데이터과학자 | 문맥 창(Context Window) 한계 설명 | 용어사전(Glossary) 충실 |

## 🔑 교차 분석 인사이트

1. **공통 패턴**: 모든 소스가 "작게 시작, 반복 개선" 강조
2. **보안**: 3/6 소스가 AI 취약점 문제 지적 (하드코딩 키·검증 누락)
3. **도구 추천 수렴**: Next.js + Supabase + Tailwind 조합 지배적
4. **간극**: 초보자용 체계적 학습 경로(커리큘럼) 부재 — 대부분 경험적 팁에 의존
