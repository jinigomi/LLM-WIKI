## 1. 데이터 구조 (Mockup)

> "이런 데이터가 필요할 것 같은데, 이게 맞나요?"

### 데이터 모델

```typescript
// 예시: 투두 앱
interface Todo {
  id: string;
  title: string;
  completed: boolean;
  createdAt: Date;
  dueDate?: Date;
}

interface Project {
  id: string;
  name: string;
  todos: Todo[];
}
```

### 검증 질문
- [ ] 이 데이터 구조가 맞나요?

## 4. 기술 선택 이유 (ADR)

> "기술 선택에는 항상 트레이드오프가 있어요. '왜 이걸 선택했는지'를 기억해두면 나중에 도움이 됩니다."

### ADR-001: [기술명] 선택

| 항목 | 내용 |
|------|------|
| **날짜** | YYYY-MM-DD |
| **상황** | [이 기술이 필요한 이유] |
| **결정** | [선택한 기술] |
| **대안** | [버린 옵션들 - 왜 안 했는지] |
| **결과** | [실제로 어떻게 될 것인가] |

**[예시: Next.js 선택]**

| 항목 | 내용 |
|------|------|
| **상황** | SEO 필요, SSR 요구, 빠른 개발 필요 |
| **결정** | Next.js (App Router) |
| **대안** | Vanilla HTML/JS (확장 어려움), React CRA (SSR 없음), Vue (생태계 작음) |
| **결과** | SSR으로 SEO 최적화, Vercel一键 배포, 파일 기반 라우팅 |
