# 의사결정

> 이 문서는 [[Engineering Thinking for Beginners]] 스킬의 핵심 산출물입니다.

---

## ADR이란?

```
"ADR은 '개발자의 노트'예요. 
3개월 뒤에 '왜 이걸 선택했지?'라고 생각할 때를 위한 거예요."
```

**기술 선택에 트레이드오프는 항상 있어요.**
- 'perfect한 선택'은 없고, '지금 상황에 가장 적합한 선택'만 있어요.

**ADR을 쓰면:**
1. 나중에 돌아와서 이해하기 쉬움
2. 다른 사람에게 설명하기 쉬움
3. 실수를 반복하지 않음

## ADR 템플릿

```markdown
## ADR-XXX: [제목]

| 항목 | 내용 |
|------|------|
| **날짜** | YYYY-MM-DD |
| **작성자** | [이름] |
| **상태** | 제안됨 | 수락됨 | 폐기됨 |

### 배경 (Context)
[문제를 어떻게 발견했거나, 왜 이 결정이 필요했는지]

### 결정 (Decision)
[무엇을 결정했는지]

### 대안 (Alternatives)
| 대안 | 장점 | 단점 | 버린 이유 |
|------|------|------|----------|
| [대안 1] | ... | ... | ... |
| [대안 2] | ... | ... | ... |

### 결과 (Consequences)
**긍정적:**
- [결과 1]

**부정적:**
- [결과 1]

### 참고 (References)
- [관련 문서나 링크]
```
