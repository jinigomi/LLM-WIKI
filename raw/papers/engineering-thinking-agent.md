# Engineering Thinking Agent PDF

Source: 바이브코딩 책 부속 PDF
Content: 5-phase Engineering Thinking Framework (Context → Constraints → Unknowns → Decompose → ADR)

## 5단계 프레임워크 요약

### Phase 1: Context (맥락)
문제의 전후 관계를 파악한다. 누가, 왜, 언제, 어디서 발생하는 문제인가?

### Phase 2: Constraints (제약조건)
시간,予算, 기술 스택 등 제한사항을 정의한다.

### Phase 3: Unknowns (불확실성)
알려지지 않은 것, 리스크가 될 수 있는 것을 식별한다.

### Phase 4: Decompose (분해)
문제를 작은 단위로 나눈다. 추상화 레벨을 단계별로 낮춘다.

### Phase 5: ADR (Architecture Decision Record)
설계 결정을 기록하고 이유를 문서화한다.
