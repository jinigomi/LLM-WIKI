설계가 잡혔다. 컴포넌트 구조, 데이터 구조, 파일 구조가 Obsidian에 기록되어 있다.

이제 Claude Code에게 이 설계를 넘기고 실제 코드를 만든다. 이 장이 "바이브코딩"의 핵심 경험이다. 내가 코드를 직접 쓰지 않는다. 설계를 읽고 구현하라고 지시한다.

코드를 저장할 폴더가 필요하다. Obsidian 볼트 밖에 별도의 폴더를 만든다.

코드 파일을 Obsidian 볼트 안에 넣지 않는 것을 권장한다. node_modules 같은 수만 개의 파일이 생기면 Obsidian이 느려질 수 있다.

Claude Code가 실행되면, 첫 번째로 할 일은 Obsidian의 설계 문서를 읽게 하는 것이다.

💡 Claude Code가 Obsidian 볼트 폴더 밖에서 실행되고 있다면, MCP를 통해 볼트의 파일에 접근한다. /mcp에서 obsidian이 Connected 상태인지 확인하자. MCP 연결이 안 되어 있다면, 설계 문서 내용을 직접 붙여넣어도 된다.

Claude Code가 설계 문서를 읽었으면, 이제 구현을 시작한다.

Ch.3의 Decomposition(분해) 원칙을 기억하자. "전부 만들어줘"는 나쁜 지시다. 한 번에 하나씩, 체크리스트의 Phase를 따라 진행한다.

이렇게 하면 Claude Code가 한 번에 모든 파일을 만든다. 뭔가 돌아가긴 하는데, 어디서 뭘 한 건지 파악하기 어렵다. 에러가 나면 어디서 시작된 건지 찾기 힘들다.

Phase 1부터 단계별로 진행한다:

Claude Code가 다음과 같은 작업을 순서대로 수행한다:

각 단계마다 Claude Code가 무엇을 하는지 터미널에 표시된다. 파일을 만들거나 수정할 때 권한 확인을 요청할 수 있다. 허용하면 진행된다.

이 화면이 보이면 성공이다: Claude Code가 "개발 서버가 실행되었습니다. http://localhost:5173 에서 확인하세요" 같은 메시지를 보여준다.

브라우저에서 http://localhost:5173을 열어보자. React의 기본 화면이 보이면 Phase 1 완료다.

💡 개발 서버를 끄고 싶으면 터미널에서 Ctrl + C를 누른다. Claude Code 자체를 끄는 게 아니라, Claude Code가 실행한 개발 서버만 끄는 것이다.

Phase 1이 완료됐으면, 다음 Phase로 넘어간다.

Claude Code가 컴포넌트를 만들고, App.jsx를 수정하고, 개발 서버에서 동작을 확인한다.

브라우저에서 확인한다: 입력창에 텍스트를 입력하고, 추가 버튼을 누르면 목록에 항목이 나타나는가? 나타나면 성공이다.

이렇게 하나씩, 하나씩 진행한다. 매번 브라우저에서 확인하고, 동작하면 다음으로 넘어간다.

코드를 이해하지 못해도 괜찮다고 했지만, 완전히 무시하는 것은 좋지 않다. 최소한 "어떤 파일이 만들어졌고, 대략 어떤 역할인지"는 파악하는 것이 좋다.

이 정도만 알아도 "어떤 파일을 고쳐야 할지" 감이 온다. 입력 폼에 문제가 있으면 TodoInput.jsx를 보면 되고, 삭제가 안 되면 TodoItem.jsx를 보면 된다.

특정 파일의 코드가 궁금하다면:

Claude Code가 코드에 한 줄씩 설명을 달아줄 것이다. 이것이 바이브코딩의 부산물 중 하나다. 코드를 직접 쓰지 않더라도, AI에게 물어보면서 점차 코드를 이해하게 된다.

구현 중에 에러가 나는 것은 정상이다. 당황하지 말자.

Claude Code의 강점은 스스로 에러를 발견하고 수정하는 자율 루프가 가능하다는 점이다(Ch.3의 Loop of Verification 원칙).

에러가 났을 때 이렇게 지시한다:

Claude Code가 에러 메시지를 읽고, 원인을 파악하고, 코드를 수정하고, 다시 빌드해서 확인하는 과정을 자동으로 진행한다.

만약 Claude Code가 같은 에러를 반복해서 못 고친다면:

💡 에러와 해결 과정은 나중에 에러로그.md에 기록한다. Ch.12에서 자세히 다룬다.

Phase 2가 끝나면 Phase 3(localStorage 연동)과 Phase 4(UI 다듬기, 배포)를 같은 패턴으로 진행한다.

배포까지 완료하면 Part 4(Ch.14~19)에서 다루므로, 여기서는 Phase 2까지만 확인되면 충분하다.

구현 도중에 시간이 없어서 멈춰야 할 때가 있다. 이때 그냥 닫지 말고, 현재 상태를 기록한다.

Claude Code의 응답을 복사해서 Obsidian의 세션 로그에 저장한다. 이 기록이 다음 세션의 출발점이 된다. 이 내용은 Ch.12에서 자세히 다룬다.
